BIM Tools

Below is a list of all our BIM Tools. A BIM tools is an aspect of BIM that on its own can benefit a project. Together these tools can have an increased benefit.

You can get to know them all, each one brings its own set of benefits to your project and when combined with others can provide even more value.

Click on a tool to reveal more information about it, or Open and Close them all.

Tap on a tool to reveal more information about it, or Open and Close.

Data Card Example

A Card that looks like this relates to a Data aspect of BIM, concentrating on improving how we capture and use information.

Model Card Example

A Card that looks like this relates to a Model aspect of BIM. Using Modelling in some form to improve our work in the real 3D world.

Common Data Enviroment

A Shared Online Enviroment in which all files and 3D models related to the project are stored. This enviroment has revision control and is accessible by all. Drawings can be annotated and shared to ensure for the best visual communication. These can be shared with the people on the project by being attached to an issue or task.

QA and Safety Checklist Capture

Using a mobile tablet to digitally fill out QA checklists and attaching them to their relating site element. Capturing safety checklists such as Quality Assurance, Site Managers Weekly HSEQ Audit Report and Site Visit Risk Assessment Sheets.

Location Tagging

The tagging of Photos, Equipment, Checklists and Tasks with location data. This can be done in two ways; the area or site can be divided into locations beforehand in order for data to be attached during the lifecycle of the project. The other method is custom tagging of keywords that are attached to the item.

Issue Tracking and Trending

Booking and logging project issues. These can be attached to site elements if the data refers to one aspect of the site. Root Causes and Issue Types can also be logged within the issue for later trending and analysis.

Daily Diary Capture

Capturing digital records of daily diaries. This auto-populates information such as weather, user details, project, date and more, also allowing the user to attach photos and text.

Electronic Data Capture

Using custom fields to capture the data we need to; programme times, personel on site, induction record sheets all done electronically using a mobile tablet.

Digital Photo Capture

Capturing and uploading Photos in high HD quality without any admin costs. Attaching locations, tags, comments and other information to photos as well as attaching photos to checklists, equipment and tasks.

Task Delegation

Allocating and organising tasks on site, the ability to create and assign tasks to members on site. These tasks can also be used to create a workflow by comments, photos, QA checklists, issues, permits, and site elements.

Comments and Communication

Improving communication using comments. Any aspect of the system can be commented on; tasks, photos, checklist. The comments are logged with a time and date it was posted. It also allows other project members the option to reply. This function could replace emails, as each series of comments are linked to a direct part of the project.

Real Time Trusted Information

All information uploading on to the system is shared, current and up to date. There is a reduced risk of duplication or deletion

Digital ITP Scaffolding

Using a work flow each QA checklist is attached to an ITP Task. This creates a ITP Scaffold Process ensuring all QA Checklists are attached to their relevant ITP Task.

Clash Detection

The process of testing for spatial overlapping within a 3D model. This can highlight design errors within one model or when comparing multiple models.

Element Data Attachment

Attaching Element Data such as QA checklists, Photos, Issues, Element Details, Tasks, Documents, Links and Model Links.

Design 3D Model

A 3D Model created from the design specification. This model can be used to take point to point measurements.

Visual 3D Model

A 3D Model created purely for visualisation purpose, no point to point accuracy.

Real Life Model Visualisation

Placing a 3D model inside a real life 3D Google Maps Enviroment

Model-Based Analysis

Deals with the qualitative costs to the tenants and the environmental costs to earth. Using values attached to the materials used within the model to create an automatic Bill of Quantities

Model Federation

Combining multiple 3D models in the same enviroment in order to compare compatibility. All models must have been created in using the same global location and grid reference

Model-Based Scheduling

A simulation of the process of works using a 3D model created using the project schedule. This simulation can be used to highlight safety or construction issues within the methodology.

Model-Based Facilities Management

Focuses on leveraging the model information to reduce the ownerâ€™s costs over the life cycle of a structure.

Digital Terrain Model

A 3D Model created from a LIDAR Scan that shows the area and height of all elements of the scan including vegetation, structures, rail track and trees.

Topographic Comparison

The comparison of two Topo Surveys or a LIDAR scan and a Topo. This can be done to create information on the movement or change of a structure, or to prove or disprove the accuracy of a Topo.

LIDAR Information

A LIDAR scan is a electronic laser scan of an enviroment. This creates thousands of digital points that together create a point cloud. This point after registration can later be used to create 3D models, compare points and for measurements.

Digital Surface Model

A 3D Model created from a LIDAR Scan that shows the area and height of the ground level (not including trees and vegetation).
